The GitHub WorkFlow


GitHub is a version control platform to save existing projects and to work on projects in a team, it uses git to provide code repositories.


Branches

The Master branch is where all the deployable code is kept. if you are working on that code you dont want to do it in master incase you break something. Instead you can create a branch from master called a feature branch which creates a seperate area to safely develop your code. The image above shows a feature branch being created from the master branch.

Commit

Once you have made changes in your feature branch you will need to save it to your local computer - this is called a commit. Once you save your work in a Commit you will get a reference number.
